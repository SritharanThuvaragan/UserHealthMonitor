# UserHealthMonitor

## Overview

This repository contains code for the **UserHealthMonitor** project, which processes raw sensor and image data to predict user health states (posture, behaviour, eye drowsiness, and lighting conditions).

## Directory Structure
```
UserHealthMonitor/
├─ dataset/
│   ├─ raw/            # Original raw data (posture CSVs, behaviour CSVs, eye images, lighting images)
│   └─ processed/      # Train/test splits and scalers generated by preprocessing
├─ ml/
│   └─ training/
│       ├─ preprocessing.py   # Full preprocessing pipeline
│       ├─ preprocess_all.py   # Wrapper to run the full pipeline
│       └─ train.py            # Train RandomForest models on processed data
├─ docs/
│   └─ implementation_notes.md
├─ requirements.txt
└─ README.md
```

## Pre‑processing

The preprocessing script (`ml/training/preprocessing.py`) performs the following steps for each dataset:
1. **Load raw data** – CSV files for posture & behaviour; image directories for eye and lighting.
2. **Clean** – Drop completely empty rows.
3. **Separate label column** (if provided) and handle missing values.
4. **Scale numeric features** with `StandardScaler` (non‑numeric data is left unchanged).
5. **Split** into an 80/20 train‑test split.
6. **Save** the feature/label CSVs and the fitted scaler under `dataset/processed/`.

Run the full pipeline with:
```powershell
# Activate virtual environment first if needed
.\.venv\Scripts\Activate.ps1
python d:/Research/UserHealthMonitor/ml/training/preprocess_all.py
```
The script outputs confirmation messages for each dataset.

## Training a Model

After preprocessing, you can train a RandomForest model for either the **posture** or **behaviour** dataset:
```powershell
python d:/Research/UserHealthMonitor/ml/training/train.py posture   # or behaviour
```
The trained model (`<split>_model.pkl`) is saved in `ml/models/`.
